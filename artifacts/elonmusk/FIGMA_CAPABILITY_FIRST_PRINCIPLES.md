# Elon Musk — First-Principles Figma Capability

## Fundamental outcome

Turn founder intent into an editable design and then into verified repository behavior without losing product truth, privacy, authority, or speed.

## Delete

- Delete the assumption that one Figma workflow fits every repo.
- Delete design-only completion claims.
- Delete invented APIs, schema, products, inventory, permissions, and provider state.
- Delete parallel component systems when existing components can be reused or wrapped.
- Delete real private data from design fixtures.
- Delete Code Connect theater without published components and exact mappings.
- Delete automatic carry-forward from design approval to code, merge, deployment, or spend.

## Simplify

```text
repo profile
+ source truth
+ synthetic data
+ reusable foundations
+ explicit states
+ exact code target
+ exact proof
```

## Bottleneck

The bottleneck is not drawing screens. It is preserving the connection between design intent, repository architecture, authority, and executable proof across seven different products.

## Fast feedback

Use Figma screenshots and metadata for visual validation, local/browser/device proof for behavior, blob/SHA binding for provenance, and exact-head runners for integration confidence.

## Automate last

Automate component synchronization and evidence refresh only after the repo profile, token model, component APIs, implementation mapping, and privacy boundary are stable. Do not automate publishing, Code Connect, merge, deployment, spending, or external communication first.
