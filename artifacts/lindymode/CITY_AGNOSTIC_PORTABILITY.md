# Lindymode — Portability That Survives Provider and Policy Change

## Durable primitives

The platform is built from jurisdiction, organization, source, program, opportunity, signal, score, and outcome. These concepts remain useful if GitHub, Supabase, Cloudflare, a specific grant program, or a specific city is replaced.

## Long-lived design choices

- UUID identities and stable slugs instead of vendor object IDs.
- Parent-child jurisdiction hierarchy instead of city-only records.
- JSON eligibility and configuration envelopes for variable program rules.
- Append-only source and score provenance rather than overwriting history.
- Explicit authority level and organization ownership.
- Service-role-only initial storage; exposure is added deliberately, not inherited accidentally.
- Versioned scoring weights so later models do not rewrite prior decisions.

## What is deliberately not generalized yet

- Automatic interpretation of legal eligibility.
- Autonomous applications or submissions.
- Public applicant records.
- Cross-jurisdiction ranking presented as objective truth.
- A universal ontology for every economic-development program.

## Portability standard

A new jurisdiction should require records, source adapters, and configuration—not a new table, route, scoring formula, or UI branch.
