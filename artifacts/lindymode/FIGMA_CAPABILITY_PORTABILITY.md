# Lindymode — Durable Figma Capability

## Durable primitives

The long-lived assets are not individual mockups. They are:

- repository-owned design profiles;
- semantic variables and stable component APIs;
- explicit state models;
- synthetic-data fixtures;
- design-to-code mappings;
- accessibility rules;
- exact evidence and rollback records.

These survive changes in Figma plans, frontend frameworks, providers, hosting, or individual products.

## Durable ordering

Inspect before creating. Variables before components. Components before screens. Design before implementation only when implementation targets are already known. Implementation before claiming completion. Runtime evidence before release claims.

## Portability rule

The portfolio protocol stays shared, but the repo profile stays local. A new repository should require a new profile and skill activation—not edits to every existing product contract.

## Provider independence

Figma may be replaced as the editing tool without losing the repository profiles, state definitions, data classifications, implementation mappings, proof model, or rollback requirements.

## What remains deliberately local

Brand language, runtime framework, component paths, test commands, data classification, design-library ownership, and authority boundaries remain repository-specific.
