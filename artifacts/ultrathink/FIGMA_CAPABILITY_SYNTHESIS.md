# ULTRATHINK — Portfolio Figma Capability Synthesis

## Core conclusion

The portfolio did not lack access to Figma tools. It lacked an enforceable translation layer between Figma operations and each repository's real product architecture.

## System layers

1. **Founder intent:** desired user outcome and business reason.
2. **Repository truth:** current framework, components, routes, schemas, providers, and tests.
3. **Design truth:** variables, components, screens, interactions, state coverage, and accessibility.
4. **Implementation truth:** exact code paths and behavior.
5. **Proof truth:** local, exact-content, exact-head, and deployed-observed evidence.
6. **Authority truth:** who may approve scope, publication, merge, deployment, data, spending, and external action.

Drift occurs when any layer is collapsed into another.

## Portfolio pattern

The shared contract standardizes operating sequence, data classification, evidence states, Code Connect gates, and handoff records. Repo-local profiles preserve framework, privacy, brand, catalog, authority, and verification differences.

## Highest-risk interfaces

- Se'kret Bip: consent, parent visibility, identity, and real-device behavior.
- Founder Control Room: false-green operational state and approval carry-forward.
- L99: canon/provenance drift and fake render/promotion claims.
- Chief AI: prototype/private-backend confusion and prompt exposure.
- Public commerce: product, catalog, pricing, checkout, and private-data leakage.
- Private commerce: Figma cloud exposure, accidental public library linkage, and deployable owner assets.

## Decision

Adopt the repo-profile architecture. Keep Code Connect conditional. Treat Figma library publication as a separate gate. Require design-to-code drift reporting. Preserve multiple evidence states rather than one completion flag.

## Next maturity steps

- reconcile existing Figma files against repo profiles;
- create repo-owned token/component scopes only where code has stable equivalents;
- implement one pilot component per UI repo;
- run visual and behavioral QA;
- publish libraries and Code Connect mappings only after ownership and plan gates are satisfied.
