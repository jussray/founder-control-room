# Bill Gates — Portfolio Figma Platform Leverage

## Bottleneck

The portfolio's design bottleneck is repeated translation: founder intent becomes a mockup, then each agent independently guesses how that mockup maps to code, privacy, data, tests, and release state.

## Standardize centrally

- skill routing and activation;
- evidence-state vocabulary;
- Figma build ordering;
- Code Connect eligibility gates;
- synthetic-data policy;
- design-to-code drift records;
- accessibility and state-coverage expectations;
- handoff and rollback fields.

## Configure locally

- framework and runtime;
- component and route paths;
- design targets;
- data classification;
- library ownership;
- prohibited content;
- test commands;
- product and authority boundaries.

## Compounding assets

Every repo implementation can improve reusable state patterns, accessibility checks, library-search decisions, fixture strategies, visual-regression methods, and handoff templates without sharing private product data.

## Platform control

Founder Control Room should track design state separately from implementation and release state. It may observe file keys, node IDs, review status, code mappings, and proof references, but it must not ingest raw private design content or turn design approval into mutation authority.

## Economic leverage

The immediate gain is reduced rework and fewer false implementation claims. Long-term leverage comes from stable repo profiles, reusable libraries with clear ownership, verified Code Connect mappings, faster onboarding, and a portfolio-wide evidence standard.

## Scale gate

Do not publish a shared library or automate synchronization until ownership, privacy classification, token semantics, component API, versioning, consumer repos, rollback, and maintenance responsibility are explicit.
